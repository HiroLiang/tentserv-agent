tentgent tui — two modes, one shell

A daemon-first, keyboard-only terminal UI. Designed as two sharply separated modes — Bootstrap when the daemon isn't reachable, Operator when it is — with no fake dashboards in between. Provider Keychain reads happen only inside an explicit per-provider action, never as a startup side-effect.

Bootstrap mode

Resolved local facts (home, daemon URL/source, env-only token sources) plus a vertical function menu: Start daemon, Daemon URL/token setup, Provider auth, Settings, Quit.

Operator mode

Function menu (Models · Adapters · Datasets · Servers · Sessions · Training · Settings · Quit) on the left; live monitoring dashboard on the right. Dashboard panels render only when the daemon returned data for them.

Bootstrap mode Daemon not reachable · no dashboard, no keychain probes · keyboard menu of next steps

What's resolved

  • Paths from env / defaults — synchronous, no I/O.
  • Daemon URL with its source (env var name vs default).
  • Per-provider env-var presence — not the value.

What's deliberately absent

  • No inventory counts. There is no daemon to count from.
  • No keychain probes. Those happen in §B4 only.
  • No "doctor" panel pretending to be a dashboard.

Menu mechanics

  • Radio glyphs ●/○ mark the focused row, not a setting.
  • ↑/↓ moves focus, ↵ activates, Esc cancels.
  • Provider auth expands inline — see §B4.

The boundary

  • Bootstrap entry & menu: no keychain reads.
  • Per-provider flow: keychain read only on Check / Set / Remove.
  • The action name on screen tells the user what's about to happen.

Three set paths

  • Set token — prompts and writes to keychain.
  • Set token (env-only) — prints an export line; user pastes into shell.
  • Remove from keychain — deletes the keychain entry, env left alone.

Per-provider

  • Same shell for openai / huggingface / anthropic.
  • Env-var name and keychain entry name are always shown.
  • Settings (§O4) reuses this exact flow.

Dashboard rendering

  • Each panel is independent. A failed sub-call collapses the panel.
  • A collapsed panel surfaces as <panel> [--] not available in Warnings.
  • Counts on menu rows mirror the dashboard so eitherside reads the same.

Auth panel

  • Reflects state the daemon resolved at startup (env > keychain).
  • Bootstrap's "not checked" disclaimer doesn't apply — the daemon owns the secret-store consumption.
  • Settings still gates per-provider Check / Set / Remove behind explicit action.

Keys

  • ↑/↓ menu focus · ↵ open
  • r refresh · : command palette
  • Tab jump pane · Esc back

Selected row

  • Caret ▌ + inverse video on the row.
  • Detail block below the table mirrors the row, doesn't duplicate it.
  • Sort indicator lives in the table header band.

Reused shell

  • Models / Adapters / Datasets / Sessions / Plans / Runs all use this layout.
  • Only the columns and detail keys change — the action band re-keys per screen.

Esc contract

  • One step back: focused table → function menu.
  • Chat (§O3) returns to this screen, not directly to the menu.

Chat surface

separate · only via Servers ▸ c chat or Sessions ▸ resume

Chat is not in the function menu. It is reached only by selecting a running server and pressing c, or by resuming a session. Esc returns to whichever screen launched it.
